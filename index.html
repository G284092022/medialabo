「Webコンテンツ作成」の成果

本テーマについて

専門科目の基礎知識を応用し、実践的な能力を習得することを目的とする。到達目標は以下の4つである。
  1. 実験の一連の流れを実践できる
  2. グループワークを円滑に実施できる
  3. モーションの解析・編集ができる
  4. Webページの作成ができる。

プロジェクト課題について

選んだものは、グルメ情報にした。プルダウンメニューからジャンルを選択し、店舗名を大きく表示し、予約するリンクなども設置した。サブジャンルが設定されていない場合などもあり、その時は次に続く検索結果も表示されなかったため、とても困った。そのため、設定されてない時は表示しないようにした。創作料理のリンクがサブジャンルと混ざっていたり、ダイニングバー・バルだと検索結果が表示されなかったので、そういった時にセンスよく表示したかったが、見栄えが悪く諦めた。

各授業回の学習事項

第1回 WWW と HTML

学習内容
Webページの作成方法、WWWについて学んだ。具体的には、リンクの構成やHTML文書の書き方、要素やハイパーリンクの追加などを実践しながら体験した。
演習の成果:演習1-4:HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

学習内容
SCCを用いて、前回作成したこのWebページの装飾をした。グラデーションを自分の想定通りに装飾するのは難しかった。リンクにオーバーマウスすると色が変わる装飾はとても気に入っている。
演習の成果:演習2-3:ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

学習内容
JavaScriptを用いて,コンソールへの表示やオブジェクトについて学んだ。1~3回までの学んだことを活かし、数当てゲームやプロジェクト課題の編集を行なった。開発者ツールでエラーの確認や、コマンドでのデバックができるのも知った。いきなりVScodeで書くとエラーのときに原因がわからない時があるので、その前にコンソールに書いてみてから実行するのが大事だと知った。
演習の成果:演習3-6:JavaScriptのオブジェクト

第4回 DOM と JavaScript

学習内容
入力フォームやボタンから、JavaScriptでのイベント処理のやりかたを学んだ。まずHTMLでのinput要素の種類から入り、JavaScriptでその値やテキストの参照方法・取り出し方を学んだ。それを用いてコンソールに表示させたり、要素のテキストを変更させたりした。それを用いて演習4-10で足し算ページを作成した。比較的難しい部分だったが、デバッグモードを使用するとどこに向けてなんの編集をするかがわかったり、作ったイベントがどこにあるか確認する方法を学んだ。それを活かし完成させることができたと思う。
演習の成果:演習4-10:足し算ページ

第5回 イベント処理

学習内容
第4回まで学んだことを中心に、イベント処理について学んだ。ボタンイベントの連携のほか、JavaScriptからHTMLの要素を新規作成、配置、削除する方法を学んだ。また要素の属性の設定も学んだことにより、画像やリンクも、HTMLでなく、JavaScriptで追加することができた。これにより、デバックモードから、HTMLの装飾を気にしながら画像を配置したり、変更できるため、装飾に困りにくいと感じた。実際プロジェクト課題で検索結果をページ上に配置するとき、レイアウトや配置の順番など変えやすかったように感じた。
演習の成果:演習5-7:DOMの演習その2

第6回を終えて

工夫点
　プロジェクト課題にて、タイトルは中央寄せにし、marginで囲った箱の余白を作って見やすくした。追加した p 要素の店舗名では、JavaScriptでフォントの調整をし、サイズを変更することができた。「プロジェクト課題について」でも述べたが、サブジャンルがないものがあり、定義されてないとでて、次の検索結果が出なかった。改善するために、サブジャンルが定義されていないものは表示しないようにした。
　このページの作成にあたり、グリッドレイアウトを多用して作成した。特に第1回〜第3回と、第4回,第5回を3分割と2分割に分けて設定したところが工夫した。同じ処理を上と下でしているのに、SCCでは分けて書いているのもあるので、無駄な文があるとは思うが、自分の実現したいことができて良かった。
苦労した点
　同様にプロジェクト課題にて、指定したジャンルから検索コードを取り出し、文字結合してurlにするイメージがついていたが、実際に値を取り出すまでに時間がかかった。理由としては、検索コードをどこから取り出そうかと悩み、検索したリンク先から取得したいが、そのリンクはコードを先に取得しないとできない...と悩んでしまったた。結果的に配列を作成し、ジャンルとコードをそれぞれ作成、ジャンルの名前が同じならそのコードを取り出すという形で落ち着いた。
強調点
　このページの背景色で、左上から右下にかけてのグラデーションになるように編集した。プロジェクト課題では、別サイトでぼかし加工した背景画像を挿入した。そのまま検索結果を配置すると見にくかったため、CSSを用いて、検索結果を線で囲み、背景色を設定した。しかし、せっかく用意した背景が見れなかったので、透けるように色設定した。第五回の時点で、検索結果を削除し、新たな検索結果を表示するように編集していたため、第6回で楽にプロジェクト課題を完成させられた。
感想
　CSSでの見た目の編集が個人的には一番大変だった。自分の思い通りにするには、どの要素を指定して、どのプロパティを使って、どのくらいの値を指定すれば実現できるかが難しかった。なかなか思い通りの場所に配置できず、妥協して配置することが多かった。また配色を考えるのも難しく、センス良くできなかった。個人的に見やすさはあると思うが、文字の色をそれぞれで変えたら見やすいかと試したりしたが、配色が悪く、結局今の色に落ち着いた。JavaScriptで、Web開発ツールを用いてコンソールに表示したり、コンソールからページ上のテキストを変更できたりするのは驚いたし楽しかった。プログラムを書いて、反映されたのを確認し、プログラムを書き直す、というのが普通だと思っていたため、コンソールで試して、うまくいってからプログラムできるのが楽しかった。
